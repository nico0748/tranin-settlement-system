# 擬似的交通系ICカード検証システム
## 開発背景
〇〇県のローカル線を運営する電鉄会社である「ウチダ電鉄」は券売機システムの刷新を計画しています。これまでは
券売機で紙の切符しか乗車券として取り扱っていませんでしたが、来年度から交通系 IC カード「UchICa」を導入し
することになりました。今回は本格的にシステム開発に着手する前にシステムの機能や動作イメージを検証するため、
コンソール画面を用いた検証用の交通系IC カード乗車システムを作成することになりました。
## システム概要
 - UchICa は初回購入時は500 円チャージされた状態で発行されます。そのため、システム起動時はチャ
ージ残高は500 円とします。
 - 下車駅の自動改札で、UchICa のチャージ残高から下車駅までの運賃を精算します。
※UchICa では、入場時（乗車時）の入場料（最低運賃）の引き去りは行いません。
 - 精算後のチャージ残高が500 円未満になると、あらかじめ登録しておいたクレジットカードで自動的に
3000 円チャージされます（自動チャージ） 。
 - 下車時、精算前のチャージ残高が運賃を下回っているときは、チャージ残額が運賃を上回るまで繰り返
し自動チャージが行われ、そののちに運賃を精算します。
 - 券売機では任意のタイミングで1000 円単位で一度に10000 円までチャージすることが可能です。決済
は登録済みのクレジットカードから行うため、現金の投入操作はありません。

## 開発における実現範囲・制約事項
今回の検証用システムでは、下車駅は両国駅固定であるものとして処理する。
また、UchICa では乗車時の運賃引き去りがないため、検証用システムでは乗車駅を選択することで、下車駅
の改札にて下車駅までの運賃の引き去りを行う設計として行う。
次ページ以降の画面仕様に従って実装してください。フォントや動作環境による表示の違いは許容するが、
数字やアルファベットは必ず半角英数字で表示する。（全角文字は使わないでください）
画面表示内容や、関数名は変更は設計書通りに実装する。
main 関数は以下のように実行することでimport した際に実行されないようにする。
```Python
if __name__ == "__main__":
main()
```
クラスを用いたオブジェクト指向のプログラミングができる人も、今回の課題では自作のクラスは用いてはならない。
